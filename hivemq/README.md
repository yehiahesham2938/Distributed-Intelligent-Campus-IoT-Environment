# HiveMQ CE backbone

This is the Phase 2 MQTT backbone for the Campus IoT environment. It
replaces the Phase 1 Mosquitto broker.

## Layout

```
hivemq/
├── conf/
│   ├── config.xml         HiveMQ listener + RBAC configuration
│   └── credentials.xml    Generated by secrets/generate_mqtt_creds.py
├── acl/
│   └── acl.xml            Generated by secrets/generate_mqtt_creds.py
└── certs/
    ├── ca.crt
    ├── hivemq_server.crt
    ├── hivemq_server.key
    └── hivemq_keystore.jks  (built from the server cert at container start)
```

## Bootstrap

1. Generate self-signed certs: `sh secrets/generate_certs.sh`
2. Build the JKS keystore for HiveMQ (the `cert-gen` compose service
   handles this automatically; the commands it runs are):

    ```sh
    openssl pkcs12 -export \
        -in secrets/hivemq_server.crt \
        -inkey secrets/hivemq_server.key \
        -name hivemq -out hivemq/certs/hivemq.p12 \
        -passout pass:changeit

    keytool -importkeystore \
        -srckeystore hivemq/certs/hivemq.p12 -srcstoretype PKCS12 \
        -srcstorepass changeit \
        -destkeystore hivemq/certs/hivemq_keystore.jks \
        -deststorepass changeit \
        -noprompt
    ```

3. Generate per-node credentials + ACLs:
   `venv/bin/python secrets/generate_mqtt_creds.py`
4. Start HiveMQ: `docker compose up -d hivemq`

## ACL isolation

`acl.xml` restricts each MQTT node to its own topic subtree
(`campus/b01/fNN/rRRR/#`) and each floor gateway to its floor subtree
(`campus/b01/fNN/#`). The `tb-integration` principal is the only
subscriber with full-campus read access; it also has write permission
for downstream commands on the `+/cmd` leaf. Any attempt by a node on
floor 01 to publish to floor 02 is rejected at the broker.
